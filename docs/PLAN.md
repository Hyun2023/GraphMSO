# 계획

## 1단계: 핵심 언어

- 그래프 모델은 관계 기반으로 유지한다: `Adj : V -> V -> Prop`를 가진
  `Graph V`.
- 그래프 클래스에 대한 보조 술어를 유지한다. 특히 `Graph.Simple`을 기본
  단순 그래프 조건으로 둔다.
- 등호, 간선, 집합 소속 관계, 명제 연결자, 1차/2차 양화사를 포함하는 MSO
  문법을 정의한다.
- 할당에 대한 타르스키 의미론을 정의한다.
- 대표적인 그래프 성질에 대한 예제를 추가한다: clique, independent set,
  dominating set, vertex cover, coloring, path-like reachability 근사, 그리고
  필요할 경우 minor 관련 인코딩.

## 2단계: 바인딩과 치환

- 이름 있는 변수가 계속 충분한지 판단한다.
- 증명 작업이 불편해지면 de Bruijn 표현이나 locally nameless 문법을 도입한다.
- 1차 변수와 2차 변수에 대한 자유 변수 술어를 정의한다.
- weakening, 자유롭지 않은 변수에 대한 할당 무관성, 알파 변환 정리를 증명한다.
- 이후 메타이론에서 필요해질 때만 capture-avoiding substitution을 추가한다.

## 3단계: 그래프 이론 API

- 유한 그래프 계층을 추가한다.
- `Graph`를 mathlib의 그래프 구조와 연결하는 mathlib 기반 브랜치를 검토한다.
- 선택한 예제들에 대해 MSO 공식과 손으로 작성한 그래프 술어 사이의 의미론적
  동치성을 증명한다.
- 재사용 가능한 MSO 명세 라이브러리를 만든다.

## 4단계: 메타이론

- 닫힌 문장에 대한 만족 관계를 정의한다.
- 그래프 클래스 위에서의 공식 동치성을 정의한다.
- 그래프 MSO에 유용한 정규형을 형식화한다.
- 표현 가능성 예제와 비표현 가능성 예제를 조사한다.
- 유한 그래프를 추가한 뒤에는 결정 가능성 및 model checking과의 연결을
  검토한다.

## 5단계: 도구화와 자동화

- 할당 갱신과 공식 평가에 대한 `simp` 보조정리를 추가한다.
- 표기법은 핵심 API가 안정된 뒤에만 추가한다.
- 대표 공식을 컴파일하는 테스트와 예제를 추가한다.
- 저장소가 최종 호스팅 위치로 옮겨지면 CI를 추가한다.
